# Content Structure Intelligence — Interview

A 4-hour system design and implementation interview. The candidate designs and builds a system that uses empirical research on website structure and LLM citation patterns to score, recommend, and optimize content structure.

## Documents

- [Research Paper](docs/research-paper.md) — Consolidated findings from two experiments on how internal link topology predicts LLM citation rates
- [Product Requirements](docs/prd.md) — Progressive requirements across three phases: scoring, recommendations, and direct optimization

## Interview Structure

| Phase | Time | Focus |
|---|---|---|
| 1. Comprehension & Scoring | 0:00–1:15 | Read research, design scoring system, implement graph construction & metrics |
| 2. Recommendations | 1:15–2:30 | Design and implement a recommendation engine that produces specific, prioritized advice |
| 3. Optimization | 2:30–3:45 | Design and implement direct structural optimization with simulation and safety constraints |
| 4. Review | 3:45–4:00 | Code walkthrough, scalability discussion, limitations |
